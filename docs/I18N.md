# I18N 说明

`apk-rebuilder` 当前前端使用轻量级内置字典，不依赖 Vue i18n。所有文案集中在 [public/modules/i18n.ts](../public/modules/i18n.ts)。

## 当前机制

- 文案字典定义在 `DICTIONARY`
- 宿主通过 iframe `INIT` / `LANG_CHANGE` 传入语言
- 页面通过 `t(key, params)` 读取文案
- 页面标题和部分静态区块会在语言切换时重新渲染

## 当前语言

目前内置：

- `zh-CN`
- `en-US`

插件 manifest 中还声明了更多名称/描述语言，见 [public/plugin-manifest.json](../public/plugin-manifest.json)。

## 新增文案的要求

新增用户可见文本时：

1. 先在 `public/modules/i18n.ts` 增加 key
2. 至少补齐 `zh-CN` 和 `en-US`
3. UI 中不要直接写死中文或英文
4. 错误提示、按钮文案、空状态文案也要走 `t()`

## Key 命名建议

按功能域组织：

- `app.*`
- `header.*`
- `upload.*`
- `pkg.*`
- `patch.*`
- `standard.*`
- `submit.*`
- `notify.*`
- `host.*`

避免：

- `text1`
- `buttonOk`
- `miscMessage`

## 参数插值

支持简单占位符：

```ts
t('apkLibrary.size', { size: '32 MB' })
```

对应文案：

```ts
'apkLibrary.size': '大小: {size}'
```

## 改语言时要注意的点

- 嵌入页要验证 `INIT` 首次语言和运行中 `LANG_CHANGE`
- 页面标题是否同步更新
- 弹窗、抽屉、按钮和空状态是否都跟随变化
- 日志或错误信息中如果是后端原文，可能不会自动翻译

## 测试建议

涉及 i18n 的改动，至少人工验证：

1. 独立页加载默认语言
2. 嵌入页收到宿主语言配置
3. 切换语言后标题、按钮、错误提示同步更新
4. 新增 key 不出现回退成 key 名本身的情况
